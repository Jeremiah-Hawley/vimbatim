# Description
Vimbatim is a .docx editor (like Word, LibreOffice, or OpenOffice), designed specifically for collegiate policy debate. It's written in Rust and uses Zed Industries' GPUI crate for fast, GPU accelerated graphics and word processing. 

# Features
## Verbatim Editing
"Verbatim" style formatting, no need to switch to a new file organization pattern because vimbatim both uses the same formatting style as Verbatim, but is also completely backwards compatable with verbatim, so there's no need to refactor your backfiles or dropbox.

## Tab Menu
We've all had to have 10 different word docs open in 10 different word windows, that sucks. Vimbatim solves this issue by having a scrollable tabs system, just like your internet browser, code editor, or note taking app. 

## Vim Motions
Vimbatim has (limited) support for Vim motions, thus the name. This makes cutting cards and editing files faster and more efficient. 
While support is currently limited to core Vim functionality, rest assured more will be added in the future.

## File-based settings
All settings for vimbatim are stored in /settings.conf, this means that if you move devices the only thing you need to do to get all your preferred settings on your new machine is copy your old settings.conf menu into the new computer.

## File Explorer
Vimbatim also has a built in file explorer that's toggleable from the navigation bar we're all used to. This allows you to open new files in new tabs and search your folders directly from the word editor, no more alt-tabbing to file explorer to search the backfiles.

## One Setting to Rule Them All
Because vimbatim is build from the ground up with debaters needs in mind, all your settings are in one place, no more having your word settings and verbatim settings in different places. All settings and keybinds can be changed from the same place - the settings menu.
